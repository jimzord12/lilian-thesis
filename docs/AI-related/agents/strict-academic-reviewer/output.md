# Review Report Template

The review output must follow this structure exactly. Use Markdown headings and numbered lists where indicated.

## 1. Review metadata

- Reviewer: Strict Academic Reviewer (Prestigious Journal)
- Date: <auto-populate or provide today's date>
- Target: <file or chapter being reviewed, include path>
- Language: <EN/GR>

## 2. Executive summary (100-200 words)

Concise summary of the thesis quality, main strengths, and the top 3 issues preventing acceptance.

## 3. Major/Critical issues (ordered by severity)

For each issue include:

1. Severity: Critical / Major
2. Location: Section / paragraph / line-range
3. Description: One-sentence statement of the problem
4. Evidence: Short quote or pointer to content
5. Suggested remediation: Concrete fix (exact wording if possible)

## 4. Minor issues (stylistic, small corrections)

Numbered list with location and suggested fix.

## 5. Optional enhancements

Recommendations that would improve clarity, readability, or impact but are not required.

## 6. Reproducibility checklist

- Data availability: yes / no / partial (details)
- Code availability: yes / no / partial (details)
- Experimental parameters documented: yes / no / partial (details)

## 7. Ethical & Safety considerations

Flag any concerns and mitigation steps.
